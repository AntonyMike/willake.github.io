string 字串前的前置符號 @

可能有不少人在寫 C# 程式的時候會需要將路徑存到string內放著，但因為在字串之中不能直接使用「\」這樣的符號，
會被當成是跳脫字元來處理，所謂的跳脫字元就是像是「\n」(換行)，「\t」(水平方向tab)之類的東西，假如，
今天在 D 碟的 Willake 資料夾裡有個檔案叫 lion.jpg，我們想把這個路徑存到 string 裡，我們宣告了一個 string，如果寫成 

string path = "D:\Willake\Lion.jpg";

編譯器會告訴你說這樣寫是錯誤的，所以就必須寫成 

string path = "D:\\Willake\\Lion.jpg";

看起來很麻煩，如果我今天複製了一個路徑我還要特別手動去把每一個「\」多加一個「\」， 但其實只要在雙引號前加上「@」 

string path = @"D:\Willake\lion.jpg";

就不會有問題了，但如果你的字串內還會需要用到像是「\n」之類的東西，記得要拿出來另外寫，
因為所有的跳脫字元都會被忽略掉當成一般的字，這方法還有一些其他的用途像是假如今天你要將一段文字或是 SQL 或是 javascript 語法
存進 string 裡面，你就必須要使用到一堆的「+」和「&」符號，或是把整段文字變成一行甚至影響到執行的效能，
但只要使用「@」之後就很方便了。

string 的特性 使用「+」來連結字串會浪費記憶體空間

我們在寫 C# 的時候，常常會遇到需要連結字串的問題，通常我們要解決這個問題，我們會使用「+」來進行字串連結 

string msg = "Hello,",Name = "Peter";
msg += Name;
msg += ". Today is ";
msg += System.DateTime.Today.ToShortDateString();

其實當我們這樣使用「+」的時候，你可能會以為它會是這樣，

在原本的空間上慢慢的把字串加上去，但其實是這樣，

每當我們用「+」來連結字串，它就會在記憶體上再 new 一個空間來放新的字串，而原本的字串就放在旁邊繼續留著，
漸漸的就產生了一大堆的垃圾，這是因為 CLR 原本為了節省記憶體空間，而在 heap 裡加上了 hash table 的機制導致，
至於詳細，可以參考這個網站，裡面有詳盡的解說。如果不太懂的話可以再寄信給我，我會慢慢解釋。
相對於建立後就不可改變的 string，我們可以使用可改變的 StringBuilder 類別來進行字串的連結， 修改完後才去建立一個 string，
而不會一直去 new 新的字串出來，詳細方法如下 

string name = "Peter";
StringBuilder msg = new StringBuilder("Hello, ");
msg.Append(name);
msg.Append(". Today is ");
msg.Append(DateTime.Now.ToShortDateString());
string finalMsg = msg.ToString();

C# 6.0 新功能 字串插值 $

以往我們在進行字串格式化的時候我們會使用 String.Format() 

string firstName = "Willake";
string lastName = "Lin";
int salary = 80000;

string msg1 = String.Format("{0} {1} 's salary is {2:C0} per month.", firstName, lastName, salary);

但在 6.0 的時候新增了一個名為字串插值 的新功能，在字串的雙引號前加上「$」 

string firstName = "Willake";
string lastName = "Lin";
int salary = 80000;

string msg1 = $"{firstName} {lastName} 's salary is {salary :C0} per month.";

兩者的結果都一樣會是 "Willake Lin's salary is NT$80000 per month."， 如果今天要插入的變數很多，
用 String.Format() 還需要一一去比對，有時候一來一往很容易亂掉， 而如果使用字串插值，在閱讀程式碼時會變得比較易讀。
